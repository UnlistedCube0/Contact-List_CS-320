# Contact-List_CS-320
This repository is documenting the work completed in CS320 @ SNHU


Q: How can I ensure that my code, program, or software is functional and secure?
A: Testing is a fundamental part of software development. While everyone wishes they could develop a piece of software perfectly secure and functional the first time, that's impossible. Both dynamic and static testing allow developers and quality engineers to ensure there are minimal defects (because no testing is perfect either) upon deployment. That's why I've begun prioritizing testing as a requirement, not an option. 


Q: How do I interpret user needs and incorporate them into a program?
A: Having intentional conversations with users and clients is a critical portion of the requirements stage. Then, during planning, I am intentional about determining whether certain requirements are feasable / possible. If there are any grey areas, I can reach back out to the customer and gain clarity. 

Q: How do I approach designing software?
A: I approach it from a holistic point of view. While going through each stage of the SDLC, I'm also considering other stages. During the requirement phase, I'm thinking about the planning phases and what will be required later. During the planning phase, I'm thinking about the development and testing phase. This allows me to minimize the tunnel-vision many developers might have when developing, because I'm not just looking for my software to work, I'm also looking for security and efficiency and thinking about how I'm going to test it later.
